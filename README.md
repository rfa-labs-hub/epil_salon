# Салон эпиляции Елены Гончаровой

Современный адаптивный сайт на чистом HTML5, CSS3 и JavaScript (ES6+).

## Структура проекта

- `index.html` — главная страница
- `css/style.css` — стили (Mobile First, до 4K)
- `js/main.js` — бургер-меню, плавная прокрутка, lazy loading
- `images/` — фото мастеров (elena.jpg, lyudmila.jpg, evgeniya.jpg, oksana.jpg, aleksandra.jpg)

## Запуск

Откройте `index.html` в браузере или используйте локальный сервер.

## Карта

На странице подключена Яндекс.Карта с приблизительными координатами Долгопрудного. Для точной метки по адресу «Лизачевское шоссе, д. 1, корп. 4» создайте свою карту в [Конструкторе Яндекс.Карт](https://yandex.ru/map-constructor/) и замените `src` у iframe в блоке «Контакты».

## SEO

В `<head>` заданы meta description и keywords. При публикации добавьте Open Graph-теги и favicon.

## Админ-панель

Панель управления контентом сайта доступна по адресу [`sugarepil.ru/admin/`](https://sugarepil.ru/admin/).

Что можно делать:

- править все таблицы цен и услуг (синхронно во всех местах сайта);
- добавлять/удалять фото в галереях «Каталог работ» и «Ламинирование ресниц»;
- заменять фото мастеров и фон главной страницы.

### Как настроить (один раз)

1. Откройте `sugarepil.ru/admin/`. Стандартный пароль: `00000000`.
2. После пароля админка попросит ключ доступа GitHub:
   - перейдите на https://github.com/settings/tokens?type=beta;
   - нажмите *Generate new token*;
   - имя — любое (например, `admin sugarepil`);
   - *Repository access* → *Only select repositories* → выберите `rfa-labs-hub/epil_salon`;
   - *Repository permissions* → *Contents* → *Read and write*;
   - сгенерируйте, скопируйте, вставьте в админку.
3. Готово. Ключ сохранится в браузере и больше его вводить не придётся.

Пароль для входа задаётся в [`admin/config.json`](admin/config.json) (хранится только SHA-256, не сам текст). После смены пароля в настройках админки файл обновляется в репозитории — один пароль на всех устройствах.

### Как это работает

Админка не использует сервер. Она напрямую коммитит изменения в репозиторий через GitHub API. После сохранения GitHub Pages пересобирает сайт за 30–90 секунд, и правки появляются для всех посетителей.

Файлы админки лежат в [`admin/`](admin/) и не влияют на основные страницы сайта.
